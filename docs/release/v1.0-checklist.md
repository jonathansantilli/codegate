# CodeGate v1.0 Release Checklist

## 1. Freeze and Versioning

- [ ] Confirm release version and tag (`vX.Y.Z`).
- [ ] Update [CHANGELOG.md](../../CHANGELOG.md) with final release notes.
- [ ] Confirm `package.json` version matches the release tag.

## 2. Quality Gates

- [ ] `npm ci`
- [ ] `npm run lint`
- [ ] `npm run typecheck`
- [ ] `npm run test`
- [ ] `npm run build`
- [ ] `npm run test:perf`

## 3. Packaging Validation

- [ ] `npm pack --dry-run` and validate published file list.
- [ ] Verify `bin.codegate` points to `dist/cli.js`.
- [ ] Smoke test tarball locally:

```bash
npm pack
npx ./codegate-<version>.tgz scan . --no-tui
```

## 4. Security and Docs

- [ ] Confirm [SECURITY.md](../../SECURITY.md) is current.
- [ ] Confirm README examples and CI/SARIF snippets still match CLI behavior.
- [ ] Validate `codegate scan . --format sarif --output codegate.sarif` and confirm SARIF opens in GitHub code scanning.

## 5. Publish

- [ ] Ensure npm trusted publisher is configured for `.github/workflows/release.yml`.
- [ ] Trigger `.github/workflows/release.yml` via tag push.
- [ ] Verify `npm publish` succeeded and package is visible on npm.

## 6. Post-Release

- [ ] Create GitHub release notes from CHANGELOG.
- [ ] Announce release and highlight upgrade path.
- [ ] Track first 24h issues and triage regressions.
